# 03 — 产品设计原则 (Product Principles)

> **Companion（伴伴）的设计准则**
> 每一个设计决策都应该能追溯到这些原则。

---

## 一、核心设计原则

### 1.1 隐私至上 (Privacy First)

> **"你的数据只属于你。"**

#### 原则

- 所有个人数据存储在本地设备
- 聊天记录分析完全在浏览器端完成
- 不上传任何用户数据到服务器
- 不追踪、不分析、不出售用户行为
- 用户可以随时导出或删除所有数据

#### 设计决策框架

| 场景 | 选择隐私 | 选择便利 | 结论 |
|------|----------|----------|------|
| 聊天分析 | 本地NLP | 云端LLM | **本地NLP** |
| 数据同步 | 不同步 | 云端同步 | **端到端加密同步** |
| 统计数据 | 不收集 | 匿名收集 | **不收集** |
| 崩溃报告 | 不收集 | 匿名收集 | **用户选择** |

#### 反模式

- ❌ 收集用户行为数据用于"产品优化"
- ❌ 第三方SDK追踪
- ❌ 默认开启数据上传
- ❌ 隐私政策中隐藏数据收集声明

---

### 1.2 温暖陪伴 (Warm Companion)

> **"像一个认识多年的老朋友。"**

#### 原则

- AI 回复有温度，不机械、不说教、不 PUA
- 每个亲友都有独特的数字形象
- 提醒不是冰冷的通知，而是带着关心的问候
- 聊天模拟还原真实的说话方式

#### 对比示例

| 场景 | 冷冰冰的 | 温暖的 | Companion 风格 |
|------|----------|--------|----------------|
| 生日提醒 | "明天是妈妈的生日" | "明天是妈妈的生日哦！" | "小雨，明天是妈妈的生日呢，要不要准备个小惊喜？" |
| 新功能 | "已添加提醒" | "好啦，我帮你记住了~" | "放心，我已经帮你记好啦~到时候会提醒你的 ❤️" |
| 错误提示 | "操作失败" | "出了点小问题" | "哎呀，好像出了点小状况，我再试试~" |
| 空状态 | "暂无数据" | "还没有添加亲友哦" | "这里还空空的呢，要不要先添加一个重要的TA？" |

#### 反模式

- ❌ "好的，已为您完成XX操作"（机械回复）
- ❌ "您已经N天没有联系TA了"（制造焦虑/PUA）
- ❌ "建议您定期更新信息"（说教）
- ❌ 过度使用感叹号和表情（谄媚）

---

### 1.3 可爱有趣 (Cute & Fun)

> **"让科技变得亲切。"**

#### 原则

- Q 版头像让每个人都有可爱的数字形象
- 1000+ 素材组合，每个人都不一样
- 成长系统让关系"活"起来
- 表情、动画让交互充满乐趣

#### 设计语言

| 元素 | 规范 | 示例 |
|------|------|------|
| 圆角 | 大圆角（12-24px） | 卡片20px，按钮12px |
| 颜色 | 暖橙色为主 | #E8734A |
| 阴影 | 柔和、低透明度 | shadow-md, alpha 0.05 |
| 字体 | 圆润友好 | 系统字体 |
| 动画 | 轻柔、有趣 | 弹性缓动 |
| 图标 | Lucide + 圆角 | 24px, rounded |

#### 反模式

- ❌ 锐利的边角（科技感太强）
- ❌ 冷色调为主（缺乏温度）
- ❌ 生硬的过渡动画
- ❌ 过于复杂的界面（失去可爱感）

---

### 1.4 智能而不侵入 (Smart but Non-intrusive)

> **"该提醒时提醒，该安静时安静。"**

#### 原则

- AI 分析在后台静默完成
- 提醒频率可自定义
- 不推送广告、不推送无关内容
- 智能但克制

#### 提醒策略

| 级别 | 频率 | 示例 |
|------|------|------|
| 重要 | 提前7天+3天+当天 | 生日 |
| 一般 | 提前3天+当天 | 母亲节/父亲节 |
| 轻量 | 仅当天 | 自定义提醒 |
| 静默 | 仅应用内展示 | 统计更新 |

#### 反模式

- ❌ 每天推送通知
- ❌ "您今天还没有打开App"（召回推送）
- ❌ 无关的推荐内容
- ❌ 强制弹窗更新提示

---

### 1.5 简单直接 (Simple & Direct)

> **"三步完成任何操作。"**

#### 原则

- 每个核心功能都必须能在 3 步内完成
- 界面信息层级清晰
- 操作路径最短
- 减少认知负担

#### 三步原则

| 功能 | 步骤 | 说明 |
|------|------|------|
| 添加亲友 | 填信息 → 选头像 → 保存 | 3步 |
| 定制头像 | 选特征 → 调颜色 → 保存 | 3步 |
| 导入聊天 | 选文件 → 等分析 → 确认 | 3步 |
| 设置提醒 | 选亲友 → 选类型 → 保存 | 3步 |

#### 反模式

- ❌ 超过5步的操作流程
- ❌ 需要阅读大量文字才能操作
- ❌ 隐藏在多层菜单中的功能
- ❌ 需要多次确认的操作

## 二、UI 设计原则

### 2.1 8pt 网格系统

所有间距、尺寸基于 8px 网格：

| Token | 值 | 用途 |
|-------|-----|------|
| space-1 | 4px | 微间距 |
| space-2 | 8px | 紧凑间距 |
| space-3 | 12px | 默认间距 |
| space-4 | 16px | 标准间距 |
| space-5 | 20px | 宽间距 |
| space-6 | 24px | 区块间距 |
| space-8 | 32px | 大区块间距 |
| space-10 | 40px | 页面边距 |
| space-12 | 48px | 安全区域 |

### 2.2 圆角设计语言

| 元素 | 圆角 | 说明 |
|------|------|------|
| 按钮 | 12px | 友好但不幼稚 |
| 输入框 | 12px | 统一风格 |
| 卡片 | 20px | 柔和包裹 |
| 头像 | 999px (圆形) | 经典圆形 |
| 标签 | 999px | 胶囊形 |
| 模态框 | 24px | 最大圆角 |
| 图标 | 2px | 微圆角 |

### 2.3 最小点击区域

所有可交互元素最小 **44×44px**，确保触摸友好。

### 2.4 对比度要求

遵循 WCAG AA 标准：
- 正文文本：**4.5:1** 对比度
- 大号文本：**3:1** 对比度
- 图标/图形：**3:1** 对比度

## 三、交互原则

### 3.1 即时反馈

| 操作 | 反馈方式 | 延迟要求 |
|------|----------|----------|
| 点击按钮 | 按压缩放 + 颜色变化 | ≤ 50ms |
| 切换页面 | 滑入动画 | ≤ 300ms |
| 保存数据 | Toast提示 | 即时 |
| 加载数据 | 骨架屏/Loading | ≤ 200ms 显示 |

### 3.2 平滑过渡

- 页面切换：滑入/滑出（300ms）
- 模态框：从底部滑入（250ms）
- Toast：从顶部滑入（200ms）
- 列表项：淡入（200ms，交错50ms）

### 3.3 手势友好

| 手势 | 操作 |
|------|------|
| 左滑 | 删除/编辑 |
| 下拉 | 刷新 |
| 捏合 | 缩放头像 |
| 长按 | 更多选项 |

### 3.4 撤销友好

所有破坏性操作都支持撤销：
- 删除亲友 → 支持撤销（5秒内）
- 清空聊天 → 二次确认
- 修改头像 → 保存前可随时返回

## 四、AI 交互原则

### 4.1 真实性

> 模拟真实的人说话方式，不是AI腔。

| 维度 | AI腔 | 真实风格 |
|------|------|----------|
| 问候 | "您好，有什么可以帮您？" | "来了~" "在呢" |
| 回复 | "好的，已为您完成" | "搞定啦" "好嘞" |
| 告别 | "祝您生活愉快" | "拜拜~" "回见" |
| 安慰 | "请不要难过" | "没事的" "会好的" |

### 4.2 一致性

每个亲友的AI分身保持一致的人格：
- 说话风格不变
- 用词习惯不变
- 情感倾向不变
- 活跃时间感知

### 4.3 情感性

AI 回复应带有适当的情感色彩：
- 开心时表达喜悦
- 难过时给予安慰
- 惊讶时表达惊叹
- 日常时保持温暖

### 4.4 边界感

AI 不会：
- 主动发起敏感话题
- 说出对方不会说的话
- 过度热情或过度冷淡
- 替代真实的社交关系

## 五、技术原则

### 5.1 隐私优先

```
设计时首先考虑：这个功能是否需要上传数据？
如果是 → 是否可以用本地方案替代？
如果必须上传 → 是否可以端到端加密？
```

### 5.2 性能优先

| 指标 | 目标 |
|------|------|
| 首屏加载 | ≤ 2秒 |
| 页面切换 | ≤ 300ms |
| 头像渲染 | ≤ 100ms |
| 聊天分析 | ≤ 5秒 |
| 冷启动 | ≤ 3秒 |

### 5.3 离线可用

所有核心功能必须在无网络环境下可用：
- 亲友管理 ✅
- Q版头像定制 ✅
- 聊天记录分析 ✅
- 聊天模拟 ✅
- 提醒系统 ✅

### 5.4 渐进增强

- 基础功能 → 所有设备
- 动画效果 → 高性能设备
- 云同步 → 有网络时
- AI增强 → 未来版本

---

## 六、内容原则

### 6.1 中文优先

- 所有界面文字使用中文简体
- 技术术语保留英文（如 API, TypeScript）
- 未来版本支持多语言

### 6.2 正能量

- 提醒文案积极温暖
- 不制造焦虑
- 不比较、不评判
- 鼓励珍惜关系

### 6.3 尊重多元

- 关系分类覆盖各种类型
- 不预设性别、年龄、职业
- 头像素材体现多样性
- 尊重每种关系的价值

---

## 七、决策框架

当遇到设计冲突时，按以下优先级决策：

```
1. 隐私 > 便利
2. 温暖 > 效率
3. 简单 > 功能丰富
4. 离线 > 在线
5. 可爱 > 专业
```

### 示例

| 场景 | 选项A | 选项B | 决策 | 理由 |
|------|-------|-------|------|------|
| 聊天分析 | 云端LLM | 本地NLP | 本地NLP | 隐私>便利 |
| 提醒文案 | "明天生日" | "明天是妈妈生日哦" | 后者 | 温暖>效率 |
| 新功能入口 | 设置页隐藏 | 首页直接展示 | 首页 | 简单>功能丰富 |
| 数据备份 | 仅云端 | 本地+云端 | 本地+云端 | 离线>在线 |

---

> **Companion 设计原则 — 每一个决策都指向同一个目标：让每一段关系都被温柔记住。**
