# 亲友伴侣

> 基于大头小人的亲友信息管理与智能提醒应用 — 产品需求文档

**版本：** v1.0  
**日期：** 2026-06-24  
**状态：** 初稿

---

## 一、需求背景

### 1.1 业务背景

在快节奏的现代生活中，人们往往拥有众多亲友，但很难记住每个人的生日、纪念日和重要节日。即便记住了日期，也常常因为忙碌而错过最佳的祝福时机。与此同时，我们与亲友的微信聊天记录中积累了大量的个性化表达方式——口头禅、语气词、表情习惯——这些构成了独特的"说话风格"，却从未被系统性地利用。

市面上的生日提醒工具（如"生日管家"、"倒数日"）功能单一，仅提供日期倒计时，缺乏情感温度和个性化体验。用户需要的不只是一个日历提醒，而是一个能唤起情感共鸣的"数字伴侣"。

### 1.2 用户痛点

> **核心痛点**
> - **遗忘**：亲友生日、母亲节/父亲节等重要日期容易被遗忘，错过祝福时机
> - **千篇一律**：即使记住了日期，祝福语也往往缺乏个性，群发感强
> - **信息分散**：亲友的基本信息（生日、喜好、关系）散落在各处，没有统一管理
> - **情感断层**：远方亲友的"存在感"逐渐降低，缺乏日常的情感连接触点

### 1.3 解决思路

将每位亲友具象化为一个可爱的"大头小人"卡通形象，把抽象的联系人列表变成一个可视化的"亲友广场"。用户点击任一头像，即可查看该人的完整信息卡片。系统会根据生日和节日自动生成提醒，并利用导入的聊天记录分析该人的说话风格，以他/她"自己的语气"发送提醒通知，让冷冰冰的日历提醒变得有温度。

`[PM 假设 — 待验证]` 用户会因为头像的个性化和说话风格模拟而产生更强的情感连接，从而提高提醒的打开率和祝福发送率。

## 三、竞品分析

### 3.1 竞品概览

| 竞品 | 产品定位 | 目标用户 | 核心亮点 | 不足 |
|------|----------|----------|----------|------|
| 生日管家 | 生日/纪念日提醒工具 | 泛社交用户 | 农历支持、星座运势 | 无个性化形象、无聊天模拟、功能单一 |
| 倒数日 (Days Matter) | 通用日期倒计时 | 效率工具用户 | 桌面小组件、多种日期类型 | 无亲友关系管理、无智能提醒 |
| 微力同步 | 通讯录+生日管理 | 商务社交用户 | 通讯录自动同步生日 | 缺乏情感化设计、无头像定制 |
| Character.ai | AI 角色对话平台 | AI 爱好者 | 角色扮演、说话风格模拟 | 非亲友管理工具、无日期提醒 |

### 3.2 竞争结论

- **核心差异点**：市面上没有一款产品同时具备"个性化头像 + 亲友信息管理 + 智能节日提醒 + 聊天风格模拟"的组合能力。亲友伴侣填补了这个空白。
- **机会**：生日提醒类产品功能同质化严重，用户留存率低。通过情感化设计（头像定制）和个性化内容（说话风格模拟）可以显著提升用户粘性。
- **风险**：聊天记录导入涉及用户隐私，需要在产品设计中明确数据安全策略和用户授权机制。

## 四、用户画像与场景

### 4.1 目标用户

| 画像 | 特征 | 使用频率 | 核心诉求 |
|------|------|----------|----------|
| 小林（25岁，上班族） | 社交圈广，亲友多，经常忘记生日 | 每周 2-3 次 | 不想再错过重要日期，希望提醒有温度 |
| 小雨（30岁，异地工作） | 远离家乡，与父母/祖父母联系少 | 每天 1 次 | 想保持与家人的情感连接，不显得生疏 |
| 阿杰（28岁，社交达人） | 喜欢有趣的互动方式，注重个性化表达 | 每天 1-2 次 | 用有趣的方式记住朋友，给朋友惊喜 |

### 4.2 核心使用场景

#### 场景一：创建亲友卡片

用户打开应用，点击"添加亲友"，填写姓名、生日、关系等基本信息，选择或自动生成一个大头小人头像，保存为卡片。卡片出现在主屏幕的"亲友广场"中。

#### 场景二：收到智能提醒

距离妈妈生日还有 3 天，应用推送通知："妈咪的生日快到了！她说：'儿子啊，妈不要什么礼物，你回来吃顿饭就好。'"——通知使用了妈妈的说话风格，让用户感到亲切和温暖。

#### 场景三：查看亲友信息

用户想给朋友送礼物但忘了他喜欢什么，打开应用点击朋友的头像，查看信息卡片中的"喜好"和"上次聊天"记录，快速回忆起关键信息。

#### 场景四：导入聊天记录

用户将与好友的微信聊天记录导出为文本文件，在应用中上传。系统分析好友的说话习惯（口头禅、常用表情、句式特征），生成个性化的提醒话术模板。

### 5.2 模块详情

#### F1 亲友广场

**业务逻辑**：应用启动后默认进入亲友广场页面，以 3-4 列网格展示所有已创建的亲友头像卡片。每张卡片显示头像、姓名、关系标签，以及即将到来的事件（如"3天后生日"）。顶部提供分组筛选标签（全部/家人/朋友/同事），右上角为搜索入口。

**交互逻辑**：点击头像卡片 → 进入该亲友的详情页（F4）；长按卡片 → 弹出操作菜单（编辑/删除/置顶）；点击"添加亲友" → 进入创建流程（F2）。支持拖拽排序自定义排列顺序。

**规则约束**：亲友数量上限 200 人；卡片排序默认按"最近事件"排序（最近生日的排最前），用户可手动切换为"自定义排序"或"姓名排序"。

**边界与异常**：空状态展示引导插画和"添加第一个亲友"按钮；网络异常时展示离线缓存数据；搜索无结果时展示友好提示。

#### F2 创建/编辑亲友

**业务逻辑**：必填字段为姓名、生日、关系标签；选填字段包括手机号、喜好、备注等。生日支持公历和农历切换，选择农历后系统自动计算每年对应的公历日期。关系标签为枚举类型：家人、朋友、同事、同学、其他。

**交互逻辑**：点击头像区域 → 进入大头小人定制页面（F3）；表单填写完成后点击"保存" → 校验必填字段 → 成功后返回亲友广场，新增卡片出现在列表末尾。编辑模式下预填充已有信息。

**规则约束**：姓名 2-20 字符，不能包含特殊符号；生日不能晚于今天；同一个手机号不能关联多个亲友。

#### F3 大头小人定制

**业务逻辑**：提供模块化的头像组件库，包括脸型（6种）、发型（12种）、眼睛（8种）、嘴巴（6种）、服饰（10种）、配饰（8种）等分类。用户可自由组合生成独一无二的卡通形象。支持 AI 照片生成：上传一张正面照，系统自动识别面部特征生成基础头像，用户在此基础上微调。

**交互逻辑**：页面分为三区域 —— 左侧为大尺寸头像预览（实时更新），中间为组件分类标签（脸型/发型/眼睛/…），右侧为当前分类下的选项列表。点击选项立即在预览中生效。底部提供"随机生成"和"从照片生成"两个快捷入口。

**规则约束**：AI 照片生成仅提取面部轮廓和肤色，不保留原始照片特征，确保卡通化效果。组件颜色支持自定义（色板选择）。生成的头像自动保存为 PNG 格式。

**边界与异常**：照片无法识别面部时，提示"未检测到人脸，请上传正面清晰照片"；组件加载失败时展示默认组件。

#### F4 信息卡片

**业务逻辑**：信息卡片是亲友详情的完整展示页，包含头像、基本信息、生日倒计时、喜好列表、说话风格摘要、模拟说话示例等模块。系统根据已导入的聊天记录自动提取说话风格关键词，展示在"说话风格"字段中。

**交互逻辑**：从亲友广场点击头像进入；页面顶部固定头像和姓名；下方为可滚动的信息区域。右上角"编辑"按钮进入编辑模式。底部提供"模拟聊天"按钮，可查看 AI 生成的更多模拟对话。

**权限逻辑**：信息卡片仅用户本人可见，不支持公开分享（隐私保护）。导出功能仅导出基本信息，不包含聊天风格数据。

#### F5 智能提醒

**业务逻辑**：系统自动识别以下事件类型并生成提醒：
- **生日**：根据亲友卡片中的生日字段，提前 1/3/7 天推送提醒
- **母亲节**（5月第二个周日）：自动关联关系标签为"母亲"的亲友，提前 3 天提醒
- **父亲节**（6月第三个周日）：自动关联关系标签为"父亲"的亲友，提前 3 天提醒
- **自定义纪念日**：用户可添加其他重要日期（结婚纪念日、相识纪念日等）

**交互逻辑**：推送通知 → 点击进入提醒详情页 → 展示亲友头像 + 个性化提醒文案 + "去祝福"快捷按钮。提醒页面支持用户编辑话术模板、调整提醒时间（提前天数）、关闭/开启特定提醒。

**规则约束**：默认提醒时间为当天 09:00，用户可自定义；每个事件最多推送 3 次提醒（提前7天、3天、当天）；已处理的提醒标记为"已完成"。

**边界与异常**：农历生日每年对应的公历日期不同，系统需自动计算并更新；如果用户关闭通知权限，展示引导开启的提示。

#### F6 聊天记录导入与 F7 风格模拟

**业务逻辑**：用户上传聊天记录文件后，系统在本地进行 NLP 分析，提取以下维度的说话风格特征：
- **高频词 TOP 20**：统计对方消息中出现频率最高的词汇
- **常用表情 TOP 10**：提取 emoji 和自定义表情的使用频率
- **句式特征**：分析常用句式结构（感叹句、疑问句、祈使句占比）
- **语气词**：识别"哈哈"、"嗯嗯"、"哦"、"啦"等语气词的使用习惯
- **消息长度分布**：统计平均消息长度，判断是"话多型"还是"惜字如金型"

**交互逻辑**：上传文件 → 显示分析进度条 → 分析完成后展示风格画像预览 → 用户确认后保存至对应亲友的说话风格字段。

**规则约束**：仅分析对方（即被标记为亲友的那个人）的消息，不分析用户自己的消息；文件格式支持微信导出的 txt 格式（带时间戳和昵称的格式）和自定义 CSV 格式。

**边界与异常**：文件格式不兼容时，展示格式说明和示例文件下载；聊天记录过少（< 50 条）时，提示"记录不足，分析结果可能不准确"；分析过程完全离线，无需网络。

## 六、系统流程

### 6.1 核心用户流程

```
打开应用
    ↓
[已有亲友?] ─否→ 引导创建第一个亲友 → 填写基本信息 → 定制大头小人头像 → 保存
    ↓是
亲友广场
    ↓
[操作选择]
    ├── 点击头像 → 查看详情卡片
    ├── 添加亲友 → 填写基本信息
    └── 导入聊天记录 → 上传文件 → 本地 NLP 分析 → 生成说话风格画像 → 保存至亲友卡片
    
收到提醒通知 → 查看个性化提醒 → 去祝福
```

### 6.2 提醒生成流程

```
日期引擎 → [检测到事件?]
              ├── 生日 → 匹配亲友卡片
              └── 节日 → 匹配关系标签
                        ↓
                  读取说话风格
                        ↓
                  AI 生成提醒文案
                        ↓
                  [风格已导入?]
                      ├── 是 → 融入说话习惯 → 推送通知
                      └── 否 → 使用默认温暖风格 → 推送通知
```
